# 🎁 E Ink 贈品庫存管理系統

線上管理贈品庫存：拿出、補貨、新增/編輯品項，所有異動自動記錄，資料永久存在 Google 試算表，多人即時共用。

## 🌐 網址

- **使用網站**：https://p60732.github.io/eink-gifts/
- **資料庫**：Google 試算表（兩張表：`庫存`、`異動記錄`）
- **原始碼**：https://github.com/p60732/eink-gifts

> 網站已內建後端網址，**任何人、任何裝置打開即可使用，不需任何設定**。

## 📖 怎麼用

### 卡片（庫存總覽）
每張卡片顯示照片、品名、目前數量、備註、最後更新時間。數量 ≤ 低庫存警示值時會標示「⚠ 低庫存」。

### 📤 拿出贈品（右下角紅色按鈕）
1. 選品項 → 顯示照片與目前庫存
2. 輸入拿出數量（超過庫存會擋下）
3. 可填備註（發給誰／活動名）與操作者
4. 確認 → 庫存減少並寫入異動記錄

### 📥 補入庫存（右下角綠色按鈕）
進貨補貨用。選品項 → 輸入補入數量 → 確認，庫存增加並記錄。

### ＋ 新增品項
填品名、數量、單位、低庫存警示、備註，並可**上傳本地照片**或**貼圖片網址**（網址優先；支援 Google Drive 分享連結，會自動轉成可顯示格式）。

### 編輯 / 刪除
每張卡片下方按鈕。刪除會二次確認。

### 🔍 搜尋
頂部搜尋框即時過濾品名。

## 📋 異動記錄（頂部「異動記錄」）
右側滑出面板，列出每筆變動（縮圖、品名、變動量、備註、操作者、時間、變動後數量）。
- **依品項篩選**：最上方下拉
- **依日期篩選**：起～迄日期範圍，「清除」還原
- **匯出**：面板右上「匯出」下載篩選後的 Excel

---

## 📊 匯出 / 匯入 Excel

### 匯出（頂部「匯出」）
下載目前庫存為 `贈品庫存_日期.xlsx`。

### 匯入（頂部「匯入」）
選擇 `.xlsx` / `.xls` / `.csv` 檔批次更新：
- 欄位需含 **品名**（必填）；其餘：ID、數量、單位、低庫存警示、備註、圖片
- **有對應 ID 的列 → 更新該品項**；ID 留空或不存在 → **新增**
- 完成後顯示「成功 N、失敗 M」

> 💡 建議流程：先「匯出」拿到範本 → 在 Excel 改數量或加新列（新列 ID 留空）→「匯入」。欄位格式完全對得上。

## 🛠 維護者資訊

### 加 / 換照片
照片放進 repo 的 `images/` 資料夾，檔名對應品項 ID（如 `images/G002.jpg`），`git push` 後約 1 分鐘生效。手機照片建議壓到單張 < 350KB。

### 改後端邏輯（`Code.gs`）
1. Google 試算表 → 擴充功能 → Apps Script，修改 `Code.gs`
2. 部署 → **管理部署作業** → 編輯（鉛筆）→ 版本選「新版本」→ 部署
3. 網址不變，前端無需改動

### 換後端網址
若重建了 Apps Script（網址改變），改 `index.html` 裡的 `DEFAULT_API_URL` 後 push；或在網站點「設定」貼新網址（僅存在該瀏覽器）。

### 資料結構
- **庫存**：ID、品名、數量、單位、低庫存警示、備註、最後更新、圖片
- **異動記錄**：時間、品項ID、品名、變動量、變動後數量、備註、操作者

## 技術

純 HTML + CSS + Vanilla JS 單檔（`index.html`），無框架。Excel 讀寫使用 [SheetJS](https://sheetjs.com/)（CDN）。後端為 Google Apps Script Web App，資料庫為 Google Sheets。部署於 GitHub Pages。
